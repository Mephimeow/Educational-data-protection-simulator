# Песочницы (Sandboxes)

Интерактивные HTML/CSS/JS симуляции для обучения кибербезопасности.

## Описание

Каждая песочница — это отдельный HTML-файл, работающий в изолированном iframe. Песочницы не требуют серверной логики — весь код выполняется на стороне клиента.

## Структура

```
sandbox/
├── phishing/           # Симуляция фишинга
├── email-client/      # Email клиент
├── wifi-hotspot/      # Wi-Fi атаки
├── social-network/     # Социальные сети
└── atm-simulator/     # Симулятор банкомата
```

## Песочницы

### 1. Phishing (порт 9081)
Интерактивная страница для демонстрации фишинговых атак.
- Поддельные формы входа
- Распознавание подозрительных URL
- Проверка на фишинговые элементы

### 2. Email Client (порт 9082)
Симуляция почтового клиента.
- Входящие письма (фишинговые и легитимные)
- Анализ заголовков писем
- Проверка отправителя

### 3. Wi-Fi Hotspot (порт 9083)
Симуляция выбора Wi-Fi сети.
- Список доступных сетей
- Демонстрация атаки "злой двойник"
- Безопасное подключение

### 4. Social Network (порт 9084)
Демонстрация социальной инженерии.
- Фейковые профили
- Запросы на общение
- Фишинговые сообщения

### 5. ATM Simulator (порт 9085)
Симулятор банкомата для обучения.
- Поиск скиммеров
- Проверка банкомата
- Безопасное использование

## Запуск

Песочницы работают в Docker контейнерах с Nginx:

```bash
docker-compose up -d sandbox-phishing sandbox-email sandbox-wifi sandbox-social sandbox-atm
```

## Коммуникация с родительской страницей

Песочницы отправляют результаты через postMessage:

```javascript
// Отправка результата из песочницы
parent.postMessage({
  type: 'sandbox-result',
  success: true,
  message: 'Вы распознали фишинговую атаку!'
}, '*');
```

## Добавление новой песочницы

1. Создать директорию в `sandbox/`
2. Добавить `index.html`
3. Добавить Dockerfile:

```dockerfile
FROM nginx:alpine
COPY . /usr/share/nginx/html
EXPOSE 80
```

4. Добавить сервис в docker-compose.yml

## Безопасность

- Песочницы работают в изолированном iframe с `sandbox="allow-scripts"`
- Нет доступа к cookies или localStorage родительской страницы
- Нет сетевых запросов к внешним ресурсам
- Вся логика — только на стороне клиента
